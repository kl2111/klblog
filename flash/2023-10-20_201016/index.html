GitHub实践总结#

1. 初始化与配置#

1.1 创建和克隆仓库#

git clone https://github.com/KL2111/my-script-collection

1.2 设置远程仓库#

git remote add origin https://github.com/KL2111/my-script-collection

1.3 SSH认证#

  • 生成SSH密钥：ssh-keygen
  • 将公钥添加到GitHub账户设置中

2. 常见操作#

2.1 提交更改#

将工作区的修改保存到暂存区并提交：

git add .
git commit -m "描述您的更改"

2.2 同步到远程仓库#

推送本地提交到远程仓库：

git push origin main

从远程仓库拉取更新到本地：

git pull origin main

2.3 暂存更改#

暂存当前的工作区更改：

git stash

恢复暂存的更改：

git stash pop

2.4 放弃更改#

放弃工作区的所有更改：

git checkout .

3. 分支管理#

3.1 查看分支#

git branch

3.2 创建和切换分支#

git branch <分支名>
git checkout <分支名>

或

git checkout -b <分支名>

3.3 合并分支#

git merge <其他分支名>

4. 错误与冲突处理#

4.1 错误提示#

根据错误信息进行相应处理，例如"unstaged changes"、“Authentication failed"等。

4.2 解决冲突#

手动编辑产生冲突的文件，删除Git自动插入的标记，并完成合并。

5. 版本控制#

5.1 语义化版本号#

按照格式 主版本号.次版本号.修订号 制定版本号，例如 1.0.0。

  • 主版本号：当你做了不兼容的API更改。
  • 次版本号：当你添加功能的方式兼容以前版本。
  • 修订号：当你进行了兼容的错误修复。

5.2 标记版本#

创建一个新的版本标签：

git tag v1.0.0

推送标签到远程仓库：

git push origin v1.0.0
